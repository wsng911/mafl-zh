# mafl-zh Prompts

> 项目：hywax/mafl 汉化版（mafl-zh）
> 技术栈：Nuxt 3 + Vue 3 + Tailwind CSS，@nuxtjs/i18n 国际化，YAML 配置驱动

---

## 功能迭代

**1. 添加服务状态历史记录**
在 mafl-zh 中为每个服务添加状态历史记录功能。记录过去 24 小时内每次健康检查的结果，在服务卡片上显示状态历史条形图（绿色=正常，红色=异常），帮助用户了解服务的稳定性趋势。

**2. 支持服务分组折叠**
在 mafl-zh 中为服务分组（Group）添加折叠/展开功能。用户可以点击分组标题折叠该分组，折叠状态保存到 localStorage，页面刷新后保持上次的折叠状态。

**3. 添加快速搜索功能**
在 mafl-zh 的顶部导航栏添加全局搜索框（快捷键 `/` 激活），支持按服务名称、描述、标签快速过滤显示的服务卡片，搜索结果实时更新，按 Esc 清除搜索。

**4. 支持服务卡片自定义布局**
在 mafl-zh 中允许用户在配置文件中为每个服务指定卡片大小（small/medium/large）和显示列数，实现更灵活的仪表盘布局，满足不同屏幕尺寸和信息密度需求。

**5. 添加天气小部件增强**
在 mafl-zh 中扩展 OpenWeatherMap 服务组件，支持显示未来 3 天的天气预报（温度、天气状况、降水概率），并支持通过配置文件设置城市名称或经纬度坐标。

## Bug 修复

**6. 修复 YAML 配置文件中中文字符导致解析错误**
在 mafl-zh 中，当用户在 YAML 配置文件中使用中文作为服务名称或描述时，偶尔出现解析错误。请检查 YAML 解析器的编码配置，确保使用 UTF-8 编码读取配置文件。

**7. 修复服务健康检查在 HTTPS 自签名证书时失败**
在 mafl-zh 中，对使用自签名 SSL 证书的服务进行健康检查时，会因证书验证失败而报错。请在配置文件中添加 `insecure: true` 选项，允许跳过 SSL 证书验证。

**8. 修复移动端服务卡片图标显示错位**
在 mafl-zh 的移动端视图中，服务卡片的图标在某些屏幕尺寸下显示错位或溢出卡片边界。请检查 Item.vue 组件的响应式样式，确保图标在所有屏幕尺寸下正确显示。

**9. 修复语言切换后页面标题未更新**
在 mafl-zh 中，通过语言切换器切换语言后，浏览器标签页的标题仍显示旧语言的文本。请在语言切换时更新 `useHead` 中的 title，确保页面标题与当前语言一致。

**10. 修复配置文件热重载后服务状态未刷新**
在 mafl-zh 的开发模式下，修改配置文件后服务列表会热重载，但服务的健康检查状态不会立即刷新，仍显示旧状态。请在配置重载后触发所有服务的状态重新检查。

## 重构

**11. 将服务健康检查逻辑提取为独立 composable**
mafl-zh 中各个服务组件各自实现健康检查逻辑，存在重复代码。请提取通用的 `useHealthCheck` composable，统一处理请求发送、状态更新、错误处理和重试逻辑。

**12. 使用 Zod 增强配置文件验证**
mafl-zh 目前对配置文件的验证较为简单，配置错误时提示不够友好。请使用 Zod 对配置文件进行严格的 Schema 验证，在启动时输出详细的错误信息（哪个字段、什么错误、如何修复）。

## 代码理解

**16. 解释 mafl 的配置驱动架构**
在 mafl-zh 中，整个仪表盘的内容完全由 YAML 配置文件驱动。请解释配置文件如何被读取和解析、服务组件如何根据配置动态渲染、配置变更如何触发页面更新，以及 Nuxt 3 的服务端渲染在这里的作用。

**17. 解释 Nuxt 3 的服务端 API 路由机制**
在 mafl-zh 中，健康检查请求通过 Nuxt 3 的 server/api 路由代理到目标服务，避免浏览器的 CORS 限制。请解释 Nuxt 3 的 server/api 目录结构、如何定义 API 路由、请求如何从客户端流转到服务端再到目标服务。
